**Fraud Detection Dashboard**
Project Overview

This project focuses on analysing financial transaction data to identify potential fraudulent activities and provide actionable insights through an interactive Power BI dashboard.

The goal is to simulate a real-world fraud analytics scenario by detecting patterns, anomalies, and high-risk segments.

**Objectives**

-Identify fraudulent transaction patterns
-Analyse fraud distribution by region, channel, and customer segments
-Build a dashboard to support decision-making
-Highlight high-risk transactions and behaviours

**Tools & Technologies**

SQL (data extraction & transformation)
Power BI (data visualisation & dashboarding)
Excel / CSV (data source)

**Dataset**

Source: Simulated financial transactions dataset (or Kaggle)
Records include:
Transaction ID
Amount
Location
Transaction type
Fraud flag

**Key Insights**

Fraud is concentrated in specific transaction channels (e.g., online payments)
High-value transactions show a higher fraud probability
Certain regions exhibit elevated fraud rates
Repeat offenders identified through customer-level analysis

**Dashboard Features**

Fraud rate KPI
Transaction trends over time
Fraud by region and channel
High-risk customer segmentation
Interactive filtering for deeper analysis

**Business Impact**

This dashboard enables:
Faster fraud detection
Improved monitoring of suspicious activities
Data-driven decision-making for risk mitigation

**How to Use**

Open the .pbix file in Power BI Desktop
Refresh data if needed
Interact with filters and visuals
